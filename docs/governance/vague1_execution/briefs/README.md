## Format des briefs

Chaque brief suit la structure :

1. **Objectif** — en 1-3 phrases
2. **Contexte** — pourquoi ça compte, quel angle mort du plan original
3. **Périmètre IN / OUT** — délimitation claire
4. **Dépendances** — ce qui doit être livré avant
5. **Fichiers à toucher** — chemins exacts
6. **Implémentation** — pseudocode + exemples minimal
7. **Acceptance criteria** — Given/When/Then ou checklist
8. **Tests requis** — unitaires + intégration
9. **Risques / pièges** — ce qui peut mal tourner
