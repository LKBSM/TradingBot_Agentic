                          PRINCIPES ARCHITECTURAUX NON-NÉGOCIABLES
═══════════════════════════════════════════════════════════════════════════════════════════════

  1. Découplage pipeline / livraison       — un seul moteur, N surfaces
  2. Idempotence des publications          — clé (signal_id + channel + target)
  3. Pas de logique business dans renderer — projection pure
  4. Pas d'appel LLM hors LLMNarrativeEng. — single LLM router
  5. Pas d'I/O direct depuis routes        — toujours via service layer (route ≤ 30 L)
  6. Reproductibilité bit-à-bit            — signal_id SHA-1 déterministe
  7. Compliance par construction           — ComplianceService injecte, jamais wrapped post-hoc
  8. Mobile-first                          — < 200 ms cache-hit sur 375px
  9. Bilingue FR + EN J1                   — tout contenu passe par i18n
 10. Un seul contrat source de vérité      — intelligence/insight_v2/ canonique
 11. Auditabilité                          — SignalStore immutable, replay = re-read
 12. edge_claim = False codifié            — interdit "edge prouvé" tant que critères non franchis
